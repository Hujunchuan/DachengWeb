# 止观 AI 网页端

止观 AI 是一个以心智训练、共修成长与数字化体验为方向的网页端原型项目。

当前仓库承载的是项目的一期官网原型：先完成页面结构、视觉表达、入口组织与基础跳转，再逐步演进到更完整的 AI 共修产品。

## 项目简介

这个网页端当前处于早期阶段，重点是把项目的整体气质、叙事方式与入口关系先建立起来。

它不是最终形态，也不是完整产品，而是一个正在成形中的起点：

- 用官网方式呈现项目愿景与方向
- 用清晰入口承接后续的 AI 与社区能力
- 为登录后私有能力和知识系统预留空间

## 当前页面结构

当前仓库包含以下主要页面：

- `index.html`：首页与总入口
- `content.html`：当前由导航中的 `AI` 入口承接
- `community.html`：当前由导航中的 `社区` 入口承接
- `download.html`：下载页
- `auth.html`：登录/注册入口页

## 导航说明

当前项目的目标导航命名为：

- `首卷`
- `AI`
- `社区`
- `下载`
- `知识库`

需要说明的是：

- 这里的 `AI` 只是当前顶栏文案，对应的仍然是现有 `content.html`
- 这里的 `社区` 只是当前顶栏文案，对应的仍然是现有 `community.html`
- 现阶段并没有进行页面文件重命名，也没有把 `content.html` 真正重构为 AI 对话页

## 页面映射说明

### 首卷

首页用于承载项目介绍、方向说明、核心视觉与主要入口。

### AI

`AI` 当前对应现有内容页入口，主要承担未来 AI 共修入口的占位与过渡角色。

### 社区

`社区` 当前对应现有社区页入口，主要承担未来用户交流与共建入口的占位与过渡角色。

### 下载

下载页用于展示移动端与应用分发相关信息。

### 登录

登录页是后续承接用户身份、私有能力与个人入口的起点。

### 知识库

知识库在产品定位上不会只是一个 GitHub 外链入口。

更准确地说：

- 当前 GitHub 仓库内容未来只会作为知识源之一
- 知识库会逐步演进为站内能力
- 后续知识库将与登录态、AI 共修和内容引用能力联动

当前仓库阶段仍以网页原型为主，知识库本身尚未独立实现。

## 当前阶段

当前阶段优先完成的是：

- 官网信息架构
- 页面入口与跳转关系
- 首页表达与品牌调性
- 交互原型的初步呈现

当前阶段暂不包含：

- 完整 AI Agent 运行层
- 登录后的私有知识库
- 正式的知识检索与引用系统
- 完整社区互动能力

## 本地预览方式

这是一个纯静态网页仓库，当前可以直接在本地浏览 HTML 文件进行预览。

常见方式：

- 直接打开 `index.html`
- 或使用任意静态服务器进行本地预览

## 后续规划

后续版本会在当前网页端基础上继续推进：

- 把 `AI` 从导航入口升级为真实的 AI 共修页面
- 把 `社区` 从展示型页面升级为更完整的交流模块
- 把 `知识库` 建设为站内私有能力，而不是简单外链
- 逐步引入登录态、成长记录与更完整的产品闭环

## 仓库说明

这个仓库目前聚焦于网页端原型本身。

它既是当前官网形态的承载点，也是后续产品演进的前端基础。
